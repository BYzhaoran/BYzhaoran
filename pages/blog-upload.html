💡 权限说明：
• 读取公开仓库：无需 Token
• 直接写入（保存到仓库）：需要 Token 拥有 `contents` 权限
• 通过 PR 发布：需要 Token 拥有 `repo` 作用域（包括 fork、分支、PR 权限）
建议使用 fine-grained PAT，在 GitHub Settings → Developer settings → Personal access tokens 创建。
